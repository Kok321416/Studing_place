О системе пользователей

Наша платформа использует современную систему аутентификации с JWT-токенами. Каждый пользователь имеет уникальные права доступа в зависимости от роли и владения контентом.

API Endpoints для пользователей

Полный набор API для работы с пользователями, включая регистрацию, авторизацию и управление профилями.

POST /api/users/register/ - Регистрация нового пользователя
Создает новый аккаунт пользователя (доступно для неавторизованных пользователей)
POST /api/users/login/ - Авторизация пользователя
Возвращает JWT-токен для доступа к защищенным эндпоинтам
GET /api/users/profile/ - Профиль текущего пользователя
Возвращает информацию о текущем авторизованном пользователе
PUT /api/users/profile/ - Обновление профиля
Обновляет информацию профиля текущего пользователя
GET /api/users/payments/ - История платежей
Возвращает историю платежей пользователя (требует авторизации)

Структура данных пользователя

Модель User:
  • email - Email пользователя (EmailField, уникальный)
  • phone - Телефон (CharField, max_length=15)
  • city - Город (CharField, max_length=100)
  • avatar - Аватарка (ImageField, опционально)
  • is_staff - Статус персонала (BooleanField)
  • is_superuser - Статус суперпользователя (BooleanField)

Система платежей

Интегрированная система для отслеживания платежей за курсы и уроки.

Модель Payment:
  • user - Пользователь (ForeignKey)
  • payment_date - Дата платежа (DateTimeField)
  • course - Курс (ForeignKey, опционально)
  • lesson - Урок (ForeignKey, опционально)
  • amount - Сумма (DecimalField)
  • payment_method - Способ оплаты (ChoiceField)
